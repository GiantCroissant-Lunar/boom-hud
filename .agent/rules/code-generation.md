# Code Generation Rules

## Principle

**Generate readable, idiomatic code that looks hand-written.**

## Rules

### 1. Use CodeBuilder

Always use `CodeBuilder` for generating source code to ensure consistent indentation:

```csharp
var code = new CodeBuilder();
code.AppendLine("namespace Generated;");
code.AppendLine();
using (code.BlockScope("public class MyComponent : View"))
{
    code.AppendLine("// Generated by BoomHud");
}
```

### 2. Preserve Component IDs

If the DSL defines an `id`, generate a field or property with that name:

```json
{
  "id": "healthBar",
  "type": "progressBar"
}
```

Generates:

```csharp
private readonly ProgressBar _healthBar;
```

### 3. Generate Comments

Include helpful comments in generated code:

```csharp
/// <summary>
/// Generated from: status-bar.hud.json
/// Component: StatusBar
/// </summary>
public class StatusBar : View
```

### 4. Handle Bindings Appropriately

**Terminal.Gui** (no native binding):
```csharp
public void Refresh()
{
    _healthBar.Fraction = (float)ViewModel.HealthPercent / 100f;
}
```

**Avalonia** (native binding):
```xml
<ProgressBar Value="{Binding HealthPercent}" />
```

### 5. Framework-Idiomatic Output

Generate code that follows each framework's conventions:

| Framework | Pattern |
|-----------|---------|
| Terminal.Gui | Imperative construction, Pos/Dim for layout |
| Avalonia | AXAML markup + code-behind, MVVM |
| MAUI | XAML + code-behind, MVVM |

### 6. Error Handling in Generators

Collect diagnostics instead of throwing:

```csharp
if (!capabilities.SupportsComponent(node.Type))
{
    diagnostics.Add(Diagnostic.Warning(
        $"Component '{node.Type}' not supported by {TargetFramework}",
        node.Id));
}
```

## Generated File Naming

| Backend | Component | Files |
|---------|-----------|-------|
| Terminal.Gui | StatusBar | `StatusBar.g.cs` |
| Avalonia | StatusBar | `StatusBar.axaml`, `StatusBar.axaml.cs` |
| MAUI | StatusBar | `StatusBar.xaml`, `StatusBar.xaml.cs` |
